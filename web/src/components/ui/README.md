# shadcn Component Library

> 20 pre-built UI primitives themed with Royal Neo-Deco CSS variables.

## Architecture

All components are generated by [shadcn/ui](https://ui.shadcn.com) and live as source files (not node_modules). They read CSS variables from `web/src/styles/globals.css` for theming.

## Installed Components

alert-dialog, avatar, badge, button, card, dialog, dropdown-menu, input, label, progress, scroll-area, select, separator, sheet, sonner, switch, table, tabs, textarea, tooltip

Most used: Button (20+ files), Badge (17+), Card, Dialog.

## Configuration

Theming is controlled by CSS variables in `globals.css`:

| Variable | Value | Purpose |
|----------|-------|---------|
| `--primary` | `#133378` | Royal Blue — buttons, headers |
| `--accent` | `#C8A951` | Metallic Gold — highlights |
| `--radius` | `1rem` | Base border radius (24px/32px Neo-Deco) |

## Adding Components

```bash
npx shadcn@latest add <component>
```

## Known Limitations

- Components use Tailwind v4 `@theme` syntax — not compatible with v3 projects
- Dark mode variables exist but are not actively used (FinalWishes is light-mode only)
