## 1. Overview

### 1.1 Architecture

FinalWishes uses a **hybrid API architecture**:

| Protocol | Purpose | Transport |
|:---------|:--------|:----------|
| **ConnectRPC (gRPC-Web)** | Estate CRUD (core domain) | Protobuf over HTTP, `@connectrpc/connect-web` client |
| **REST (JSON)** | PII Vault, Lockbox, Guidance, Payments, Media, Guardian, Capsules, E-Signatures | Standard HTTP + JSON |

The Go API (Chi router) serves both protocols from a single Cloud Run service. ConnectRPC handlers are registered at the standard Connect path prefix; REST endpoints live under `/api/v1/`.

### 1.2 Authentication

- **Provider:** Firebase Auth (Admin SDK server-side verification)
- **Method:** Bearer token in `Authorization` header
- **MFA:** TOTP required for sensitive operations
- **Unauthenticated endpoints:** `/health`, `/healthz`, `GET /api/v1/payments/tiers`, `POST /api/v1/payments/webhook`, `POST /api/v1/capsules/deliver`

```
Authorization: Bearer <firebase_id_token>
Content-Type: application/json
```

### 1.3 Rate Limiting

- **Limit:** 100 requests per 60 seconds per IP
- **Ban duration:** 10 minutes on exceed
- **Implementation:** Application-level (Cloud Run direct, no Global External ALB)

### 1.4 CORS

**Production origins:**
- `https://finalwishes-prod.web.app`
- `https://finalwishes-prod.firebaseapp.com`
- `https://finalwishes.app`

**Local dev origins:**
- `http://localhost:3000`
- `http://localhost:5173`

**Allowed headers:** `Accept`, `Authorization`, `Content-Type`, `X-Request-ID`, `Connect-Protocol-Version`

### 1.5 Error Response Format

All REST endpoints return errors in this shape:

```json
{
  "error": {
    "code": 400,
    "message": "Human readable message"
  }
}
```

ConnectRPC errors follow the standard Connect error protocol with `Connect-Error-Code` and `Connect-Error-Message` headers.

### 1.6 Timeouts

- **Server read timeout:** 5 minutes (extended for video uploads)
- **Server write timeout:** 6 minutes (extended for video uploads)
- **Request timeout middleware:** 6 minutes

## 3. ConnectRPC -- Estate Service

**Proto definition:** `proto/estate/v1/estate.proto`
**Package:** `estate.v1`
**Client transport:** `@connectrpc/connect-web` (gRPC-Web)
**Path prefix:** Auto-generated by `estatev1connect.NewEstateServiceHandler`

All ConnectRPC methods require Firebase Auth (enforced via auth middleware on the handler path).

### 3.1 Service Definition

```protobuf
service EstateService {
  // Core Estate Management
  rpc ListEstates(ListEstatesRequest) returns (ListEstatesResponse);
  rpc RegisterEstate(RegisterEstateRequest) returns (RegisterEstateResponse);
  rpc GetEstateMetadata(GetEstateMetadataRequest) returns (GetEstateMetadataResponse);
  rpc ListAssets(ListAssetsRequest) returns (ListAssetsResponse);
  rpc AddAsset(AddAssetRequest) returns (AddAssetResponse);
  rpc ListBeneficiaries(ListBeneficiariesRequest) returns (ListBeneficiariesResponse);
  rpc AddBeneficiary(AddBeneficiaryRequest) returns (AddBeneficiaryResponse);

  // The Vault (Legal Evidence)
  rpc ListVaultDocuments(ListVaultDocumentsRequest) returns (ListVaultDocumentsResponse);
  rpc GenerateUploadUrl(GenerateUploadUrlRequest) returns (GenerateUploadUrlResponse);

  // Memoirs (Videos & Photos)
  rpc ListMemoirs(ListMemoirsRequest) returns (ListMemoirsResponse);
  rpc UploadMemoir(UploadMemoirRequest) returns (UploadMemoirResponse);

  // Obituary Service
  rpc GetObituary(GetObituaryRequest) returns (GetObituaryResponse);
  rpc SaveObituary(SaveObituaryRequest) returns (SaveObituaryResponse);

  // AI Guidance Engine
  rpc GetAIInsight(GetAIInsightRequest) returns (GetAIInsightResponse);

  // Governance & Security
  rpc GetGovernanceSettings(GetGovernanceSettingsRequest) returns (GetGovernanceSettingsResponse);
  rpc ListNotifications(ListNotificationsRequest) returns (ListNotificationsResponse);
}
```

## 4. PII Vault (REST)

**Storage:** Cloud SQL (PostgreSQL 15) with Cloud KMS envelope encryption (AES-256-GCM)
**Security:** Per-estate AAD (Additional Authenticated Data), SOC 2 audit logging

### 4.1 Store User PII
```
POST /api/v1/vault/user-pii
```
**Auth:** Firebase Bearer token

**Request Body:**
```json
{
  "estate_id": "string (required)",
  "ssn": "string (optional)",
  "date_of_birth": "string (optional)"
}
```
At least one PII field (`ssn` or `date_of_birth`) is required.

**Response (200 OK):**
```json
{
  "id": "string",
  "success": true,
  "message": "PII stored securely"
}
```

### 4.2 Retrieve User PII
```
GET /api/v1/vault/user-pii?estate_id=xxx&full=true
```
**Auth:** Firebase Bearer token

**Query Parameters:**
- `estate_id` (required) -- Estate identifier
- `full` (optional, `"true"`) -- Return full PII values (default returns masked, e.g., SSN last 4 only)

**Response (200 OK):**
```json
{
  "id": "string",
  "ssn_last4": "1234",
  "ssn": "123-45-6789",
  "date_of_birth": "1970-01-01"
}
```
`ssn` field only included when `full=true`. `ssn_last4` always included when SSN is stored.

### 4.3 Store Asset PII
```
POST /api/v1/vault/asset-pii
```
**Auth:** Firebase Bearer token

**Request Body:**
```json

  "asset_id": "string (required)",

  "account_number": "string (optional)",
  "routing_number": "string (optional)",
  "vin": "string (optional)"
}
```
At least one PII field required.

**Response (200 OK):**
```json

```

### 4.4 Retrieve Asset PII


**Query Parameters:**

## 5. Digital Lockbox (REST)

**Storage:** Firestore at `estates/{estateId}/lockbox/{itemId}`
**Encryption:** Cloud KMS envelope encryption with per-estate+item AAD and per-field nonce derivation
**Audit:** SOC 2 compliant -- every credential operation is audit-logged (blocking). Unaudited access is rejected.

### 5.1 Store Credentials
```
POST /api/v1/lockbox/store-credentials
```
**Auth:** Firebase Bearer token (must be estate owner or admin)

**Request Body:**
```json
{
  "estateId": "string (required)",
  "itemId": "string (required)",
  "username": "string (optional, stored plaintext)",
  "password": "string (optional, encrypted)",
  "pin": "string (optional, encrypted)",
  "notes": "string (optional, encrypted)"
}
```
At least one sensitive field (`password`, `pin`, or `notes`) is required.

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Credentials encrypted and stored securely"
}
```

### 5.2 Retrieve Credentials

**Auth:** Firebase Bearer token (must be estate owner or admin)

**Request Body:**
```json
{
  "estateId": "string (required)",
```

## 13. Time Capsules (REST)

Deferred message delivery powered by Cloud Tasks. Capsules are written to Firestore, then a Cloud Task is created for delivery at the scheduled time. Delivery triggers the Firebase SendGrid extension via the `mail` collection.

### 13.1 Schedule Capsule Delivery
```
POST /api/v1/capsules/schedule
```
**Auth:** Firebase Bearer token

**Request Body:**
```json
{
  "estateId": "string (required)",
  "capsuleId": "string (required)"
}
```

**Delivery types (read from Firestore capsule doc):**
- `scheduled_date` -- Cloud Task with `ScheduleTime`
- `anniversary` -- Cloud Task for next occurrence (re-enqueues after each delivery)
- `on_death` / `on_settlement` -- No task; triggered by estate status listener

### 13.2 Cancel Scheduled Capsule
```
POST /api/v1/capsules/cancel
```
**Auth:** Firebase Bearer token

**Request Body:**
```json
{
  "estateId": "string (required)",
  "capsuleId": "string (required)"
}
```

### 13.3 Deliver Capsule (Internal)
```
POST /api/v1/capsules/deliver
```
**Auth:** None (called by Cloud Tasks with OIDC token + Cloud Tasks headers)

**Request Body (from Cloud Tasks):**
```json
{
  "estateId": "string",
  "capsuleId": "string",
  "action": "deliver"
}
```

This is an internal endpoint. It should not be called by clients directly.

## 15. Infrastructure

### 15.1 Backend Stack

| Component | Technology |
|:----------|:-----------|
| Language | Go 1.26 |
| Router | Chi v5 |
| RPC | ConnectRPC (Protobuf) |
| Hosting | Cloud Run (single container) |
| Database (PII) | Cloud SQL PostgreSQL 15 |
| Database (Real-time) | Firestore |
| Object Storage | Cloud Storage |
| Encryption | Cloud KMS + AES-256-GCM envelope encryption |
| Auth | Firebase Admin SDK |
| AI | sirsi-ai (Claude Opus primary, Genkit/Gemini fallback) |
| Scheduling | Cloud Tasks + Cloud Scheduler |
| Email | Firebase SendGrid extension (Firestore trigger) |
| Payments | Stripe v81 |
| Logging | zerolog (structured JSON) |

### 15.2 Security Middleware Stack

Applied to all requests in order:
1. `middleware.RequestID` -- Unique request ID
2. `middleware.RealIP` -- True client IP
3. `middleware.Logger` -- Request logging
4. `middleware.Recoverer` -- Panic recovery
5. `middleware.Timeout` -- 6-minute timeout
6. `ratelimit.Middleware` -- 100/60s per IP
7. `cors.Handler` -- Origin allowlist
8. `appmw.SecurityHeaders` -- CSP, HSTS, X-Frame-Options

### 15.3 Conditional Service Initialization

Services degrade gracefully based on available infrastructure:

| Condition | Disabled Endpoints |
|:----------|:-------------------|
| No Cloud SQL connection | PII Vault (`/api/v1/vault/*`) |
| No Cloud KMS | Lockbox (`/api/v1/lockbox/*`) and Vault |
| No Firestore | All estate, guidance, guardian, capsule, memoir, transcription, tier-gating endpoints |
| No Cloud Tasks | Capsule scheduling (`/api/v1/capsules/*`) |
| No Stripe key | Payment endpoints (`/api/v1/payments/*`) |
| No YouTube API | Memoir video endpoints (`/api/v1/memoirs/*`) |
| No sirsi-ai | Document Intelligence disabled; Guidance uses deterministic mode |
